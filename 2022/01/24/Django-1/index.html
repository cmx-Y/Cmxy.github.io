URL

Uniform Resource Locator

定义：统一资源定位符

作用：表示互联网上某个资源的地址

统一格式：

protocol://hostname[:port]/path[?query][#fragment]

  • protocol - 协议
  • hostname - 主机名
  • path - 文件路径
  • ?query 参数
  • #fragment 锚点，定位文章的某个位置

eg：

127.0.0.1:4000

访问本机的4000端口

https://www.bilibili.com/video/BV1vK4y1o7jH?p=4

通过p=4，会定位到视频P4

https://docs.djangoproject.com/en/4.0/#the-template-layer

通过the-template-layer，会定位到文档中相应位置

Django如何处理URL请求

  • 由settings.py中的ROOT_URLCONF找到主路由文件
  • 加载主路由文件中的urlpatterns
  • 依次匹配
  • 匹配成功则调用对应的视图函数
  • 匹配失败 - 返回404

视图函数

  • 接收浏览器的请求，然后返回响应
  • 具体的实现是，接收HttpRequest对象，返回HttpResponse对象

eg:

path函数

  • path(route, views, name = None)
    • route : 字符串
    • views : 视图函数
    • name : 地址的别名

path转换器：

  • 语法：<转换器类型：自定义名>

  • 作用：若转换器匹配成功，则将匹配到的数据作为参数传递给视图函数

    eg:

    path('page/<int:pg>', views.pagen_view)

re_path()

正则path

  • 语法：(?Ppattern)，若匹配成功，则将数据作为参数传递给视图函数

    eg：

    re_path(r'^(?P<x>\d{1,2})/(?P<op>\w+)/(?P<y>\d{1,2})$', views.cal_view)
